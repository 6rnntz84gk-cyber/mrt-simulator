🚇 Singapore MRT Network Simulator

Native SwiftUI iPad Application

An authentic replica of Singapore's MRT Operations Control Centre featuring real-time train simulation, passenger demand modeling, and interactive network visualization. Built with SwiftUI for iPad with MapKit integration and Canvas-based schematic mapping.

🚀 Getting Started

Prerequisites:

  • Xcode 14.0 or later
  • iOS 15+ deployment target
  • iPad Air 2 or later (or simulator)

Installation:

# Clone the repository git clone https://github.com/6rnntz84gk-cyber/mrt-simulator.git # Navigate to project directory cd mrt-simulator # Open in Xcode open MRTSimulator.xcodeproj

Running:

  1. Select iPad Pro (12.9-inch) simulator or your iPad device
  2. Press Cmd+R to build and run
  3. The app will launch in landscape orientation
  4. Use the line selector to switch between NSL and NEL
  5. Tap stations to view details and demand profiles
  6. Use Pan/Zoom on the schematic map
  7. Tap "Geographic Map" to see real coordinates

📈 Simulation Details

North South Line (NSL)

Fleet Size: 25 active trains

Route: Jurong East (NS1) → Marina South Pier (NS16 / extension)

High Demand Stations: Jurong East, Orchard, Dhoby Ghaut, City Hall, Raffles Place, Woodlands

Headway: ~2-3 minutes during peak hours

North East Line (NEL)

Fleet Size: 20 active trains

Route: Punggol (NE1) → Jurong East (NE16)

High Demand Stations: Punggol, Serangoon, Dhoby Ghaut, Outram Park, Jurong East

Headway: ~2-3 minutes during peak hours

Demand Modeling

  • Peak Hours (7-9 AM, 5-7 PM): 1.5x demand multiplier
  • Off-Peak: 0.5x demand multiplier
  • CBD Stations: Higher weekday morning and evening demand
  • Residential Stations: Morning outbound, evening inbound
  • Interchange Stations: Elevated demand throughout the day

📝 Code Architecture

MVVM Pattern

Models: Station, Train, DemandProfile, LineDefinition

ViewModels: SimulationViewModel (manages state), MapViewModel (manages views)

Views: SwiftUI components with Canvas for rendering

Managers: DataManager (JSON loading), TrainSimulationEngine (train logic)

Real-Time Updates

The TrainSimulationEngine uses a Timer (1.0s interval) to update train positions, passenger loads, and demand profiles. All updates are dispatched to the main thread for UI synchronization.

Data Flow

DataManager (loads JSON) ↓ SimulationViewModel (manages state) ↓ TrainSimulationEngine (updates trains) ↓ ContentView & Map Views (render UI)
